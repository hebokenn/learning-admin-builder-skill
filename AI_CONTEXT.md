# AI 快速上下文：Learning Admin Builder

这份文件给 AI 先读。目标是让 AI 在最短时间内掌握整个仓库的用法，然后帮助用户把学习、培训、认证、带教、课程进度项目做成可用的后台管理网站。

## 一句话定位

把“培训项目 + 学员数据 + 进度规则 + 管理流程”转成一个信息密集、克制、可扫描、可维护的后台管理系统，而不是营销展示页。

## 你应该先做什么

1. 先检查目标项目现有资料：代码仓库、README、AGENTS、handoff、Excel / CSV 样例、业务文档、现有页面。
2. 如果资料不足，先问需求；每次最多问 5 个关键问题。
3. 不要一开始写代码。先输出需求、页面、数据、权限、导入、API、脚本、验收计划。
4. 如果用户明确要求执行，再按目标仓库现有技术栈分阶段实现。

## 人机分工

人只负责业务确认和授权：目标、脱敏样例、第一版范围、账号/费用/部署/生产密钥。AI 负责尽量多的执行：读取文件、分析表格、整理方案、写代码、运行检查、生成部署前清单和交接记录。

## 不可破坏的原则

- 不上传、不复制、不暴露真实学员数据、私有导出、数据库、`.env`、账号密码、服务器地址、API key、生产邮件配置。
- 不把某个旧项目的字段、阶段、评分规则、页面文案硬套到新项目。
- 不默认存在预约、评分、通知、自动周报、证书、第三方同步；先确认是否需要。
- 不做营销首页。第一屏应直接服务管理动作：风险、待办、进度、导入、报表。
- Excel / CSV 导入必须有预览、字段映射、行级错误、冲突处理、导入日志。
- 后台管理必须考虑用户、角色、权限、日志、备份、恢复、导出权限和误操作处理。

## 需求收集最小清单

必须弄清楚：

- 项目目标：为什么做网站，成功标准是什么。
- 使用角色：管理员、培训师、导师、学员、经理、外部合作方。
- 角色权限：谁能看、谁能改、谁能导入、谁能导出。
- 学习结构：阶段、任务、里程碑、批次、自由进度。
- 数据来源：手工录入、Excel、CSV、平台 API、已有数据库。
- 学员唯一键：工号、邮箱、手机号、外部系统 ID，是否稳定且允许使用。
- 状态规则：未开始、进行中、完成、逾期、未通过、高风险。
- 报表和通知：谁看、多久一次、从哪里触发。
- 部署和维护：谁负责运行、备份、账号、故障处理。

## 推荐输出格式

当用户让你“帮我做成这种网站”时，先输出：

```text
1. 我已理解的项目目标
2. 还需要确认的问题
3. 第一版建议模块
4. 暂不建议第一版做的模块
5. 页面清单
6. 数据表和字段草案
7. 导入和冲突规则
8. 权限设计
9. API / 自动化脚本清单
10. 验收和部署计划
```

执行型 agent 还应先填写 `templates/AI执行任务单.md` 和 `templates/AI执行前项目审查清单.md` 的内容摘要，再开始改代码。

当用户让你“开始实现”时，按小步交付：

```text
1. 读取目标项目规则和现有结构
2. 建立或调整数据模型
3. 做页面骨架和导航
4. 接入真实数据/API
5. 做导入预览和日志
6. 做权限和后台设置
7. 做报表、通知、可选模块
8. 本地验证、浏览器验证、安全检查
9. 更新目标项目交接记录
```

## 默认模块

第一版通常包含：

- 登录和角色
- 首页看板
- 学员列表
- 学员详情
- 培训项目 / 阶段 / 任务
- 进度记录
- Excel / CSV 导入预览
- 跟进事项
- 基础报表
- 后台设置

可选模块，确认后再做：

- 预约
- 评分
- 证书
- 邮件或站内通知
- 自动周报
- 第三方平台同步
- 多组织 / 外部合作方门户

## UI 默认方向

- 后台工具，不是宣传页。
- 浅色、克制、低饱和度、稳定布局。
- 第一屏显示指标、风险、待办、最近导入状态。
- 表格、筛选、批量操作优先。
- 手机端表格可改成卡片或横向滚动，但不能内容重叠。
- 状态颜色全站统一：正常、落后、高风险、完成、待处理、失败。

## 数据和后台管理重点

后台不是只有页面。必须覆盖：

- 用户管理、角色权限、账号禁用和重置密码。
- 学员、课程、任务、进度的新增、编辑、归档。
- 导入预览、导入日志、错误行、回滚或恢复说明。
- 操作审计：谁在什么时候改了什么。
- 报表导出权限和导出记录。
- 通知发送记录和失败重试。
- 数据库备份、恢复演练、健康检查。

## 完成定义

一个方案或实现只有满足这些条件，才算可交付：

- 用户知道第一版要做什么、不做什么。
- 页面、数据、权限、API、导入、报表、维护都闭环。
- 可选模块没有被默认硬塞。
- 有本地验证和浏览器验证计划。
- 有上线部署和备份说明。
- 没有公开任何真实隐私和生产配置。
