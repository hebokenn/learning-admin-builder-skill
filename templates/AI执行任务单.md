# AI 执行任务单

把这份任务单交给能操作项目的 AI。AI 填完并执行，人只确认业务和授权事项。

## 1. 项目读取

AI 需要先检查：

- [ ] `README.md`
- [ ] `AGENTS.md`
- [ ] handoff / memory 文件
- [ ] `package.json`
- [ ] 数据库或 schema 文件
- [ ] 样例 Excel / CSV
- [ ] 现有页面和 UI 组件
- [ ] 当前 git 状态

AI 输出：

```text
项目规则：
技术栈：
运行命令：
已有模块：
样例数据：
风险：
```

## 2. 只问关键问题

问题必须影响以下至少一项：

- 学员唯一键。
- 角色权限。
- 导入冲突规则。
- 部署和数据安全。

## 4. 页面、数据、权限

AI 输出：

- 页面清单。
- 数据字段清单。
- 权限矩阵。
- API 与脚本清单。
- 导入和错误处理。
- 后台管理和维护项。

## 5. 执行阶段

| 阶段 | AI 要做什么 | 验证 |
|---|---|---|
| 1 | 页面骨架和导航 | 页面能打开 |
| 2 | 数据模型和假数据 | 列表、详情有数据 |
| 3 | API 和数据库 | lint / build |
| 4 | 导入预览和日志 | 错误行可见 |
| 5 | 权限和后台设置 | 无权限被拦截 |
| 6 | 报表和可选模块 | 报表可生成 |
| 7 | 部署前准备 | 环境变量和安全检查 |

## 6. 人必须授权的事项

- [ ] 使用真实学员数据。
- [ ] 填写生产密钥。
- [ ] 创建或连接生产数据库。
- [ ] 正式部署上线。
- [ ] 发送真实邮件或通知。
- [ ] 删除或覆盖生产数据。

## 7. 交付证据

AI 最终必须输出：

- 改了哪些文件。
- 跑了哪些验证。
- 哪些页面检查过。
- 是否有未解决问题。
- 是否有真实数据或密钥风险。
