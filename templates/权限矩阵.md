# 权限矩阵

设计后台管理系统时，由 AI 先根据需求生成权限矩阵。人只审批异常权限、高风险操作、导出范围和外部合作方数据隔离。页面按钮和 API 都要跟权限矩阵一致。

给 AI 的指令：

```text
请根据角色、页面、数据字段和 API 清单生成权限矩阵。
请标出高风险权限，例如导入、导出、删除、修改角色、发送真实通知、查看敏感字段。
不要默认所有管理员都能导出全部数据，导出范围必须按角色限制。
```

## 页面权限

| 页面 / 模块 | 超级管理员 | 培训管理员 | 培训师 / 导师 | 学员 | 经理 / 只读 | 外部合作方 |
|---|---|---|---|---|---|---|
| 首页看板 | 全部 | 负责项目 | 负责学员 | 自己 | 团队汇总 | 自己组织 |
| 学员列表 | 全部 | 负责项目 | 负责学员 | 无 | 团队汇总 | 自己组织 |
| 学员详情 | 全部 | 负责项目 | 负责学员 | 自己 | 团队只读 | 自己组织 |
| 课程结构 | 管理 | 管理 | 只读 | 只读 | 只读 | 只读 |
| 任务进度 | 管理 | 管理 | 更新负责学员 | 查看自己 | 团队只读 | 自己组织 |
| 导入 | 全部 | 负责项目 | 无 | 无 | 无 | 可选 |
| 报表 | 全部 | 负责项目 | 负责学员 | 自己 | 团队汇总 | 自己组织 |
| 设置 | 全部 | 部分 | 无 | 无 | 无 | 无 |

## 操作权限

| 操作 | 谁能做 | 是否需要二次确认 | 是否记录日志 | 备注 |
|---|---|---|---|---|
| 新增学员 | 管理员、培训管理员 | 否 | 是 |  |
| 编辑学员基础信息 | 管理员、培训管理员 | 重要字段需要 | 是 |  |
| 归档学员 | 管理员、培训管理员 | 是 | 是 | 不建议硬删除 |
| 导入数据 | 管理员、培训管理员 | 是 | 是 | 必须先预览 |
| 导出报表 | 管理员、培训管理员、经理 | 视敏感度 | 是 | 导出范围受角色限制 |
| 更新进度 | 管理员、培训管理员、导师 | 否 | 是 | 导师只限负责学员 |
| 修改课程结构 | 管理员、培训管理员 | 是 | 是 | 会影响进度计算 |
| 发送通知 | 管理员、培训管理员 | 是 | 是 | 记录接收人和结果 |
| 修改角色 | 超级管理员 | 是 | 是 | 高风险操作 |

## API 权限

| API | 允许角色 | 范围限制 | 敏感字段处理 |
|---|---|---|---|
| `GET /api/dashboard/summary` | admin、trainer、manager | 按角色过滤项目/团队 | 不返回个人敏感备注 |
| `GET /api/learners` | admin、trainer、manager | trainer 只看负责学员 | 手机、邮箱按需要脱敏 |
| `PATCH /api/learners/:id` | admin、trainer | trainer 只改负责学员 | 记录修改前后 |
| `POST /api/import/preview` | admin、training_admin | 负责项目 | 不落库 |
| `POST /api/import/commit` | admin、training_admin | 负责项目 | 写导入日志 |
| `POST /api/reports/export` | admin、training_admin、manager | 限制导出范围 | 记录导出人 |

## 验收

- [ ] 页面看不到无权限按钮。
- [ ] 直接访问无权限 API 会被拒绝。
- [ ] 列表数据按角色过滤。
- [ ] 导出范围按角色过滤。
- [ ] 敏感字段不会被无权限角色看到。
- [ ] 高风险操作有二次确认和日志。
