# 适配执行手册

完成需求澄清后、开始实施前，按这个手册把新学习项目映射成网站。

## 1. 判断建设类型

选择一种路径：

- 从零新建：使用用户偏好的技术栈，或推荐简单可维护的后台应用。
- 改造已有项目：保留现有框架、路由、样式、认证、数据库和组件习惯。
- 复用旧经验：只复用产品架构和工作流思想，不复制私有代码或私有数据。

## 2. 定义第一版模块

给每个模块标记：

- 第一版必做。
- 第一版可选。
- 明确不做。

模块包括：

- 首页看板。
- 数据导入。
- 学习结构。
- 任务 / 进度看板。
- 学员详情。
- 跟进事项。
- 报表。
- 通知。
- 预约。
- 评分。
- 证书。
- 趋势分析。
- 后台设置。

每个启用模块都必须有：

- 页面或视图。
- API 或数据访问路径。
- 数据表或数据来源。
- 角色权限。
- 验收标准。

## 3. 映射数据模型

从新项目业务出发，不从旧 schema 出发。常见实体：

- user
- learner
- role
- program
- cohort
- stage
- task
- progress
- manual_override
- follow_up
- import_job
- import_error
- report_run
- notification
- optional booking
- optional rubric
- optional score
- optional certification

先定义唯一键和关系，再写代码。只有用户确认稳定且允许使用时，才把邮箱当唯一键。

## 4. 设计导入

对每个导入文件或 API，定义：

- 来源类型。
- 样例结构。
- 必填字段。
- 字段映射。
- 状态标准化规则。
- 新增、更新、删除、冲突规则。
- 预览输出。
- 导入日志字段。
- 回滚或恢复预期。

没有样例文件时，只能先做手工录入、假数据或字段占位，不能编造真实导入规则。

## 5. 设计导航

默认管理员导航：

- 总览。
- 学员。
- 课程 / 阶段。
- 任务 / 进度。
- 跟进。
- 导入。
- 报表。
- 设置。

默认学员导航：

- 我的进度。
- 可选预约。
- 可选成绩。
- 设置。

没有实现的页面不要出现在导航里。导航标签要短，并使用新项目业务语言。

## 6. 安全实施

改造已有仓库前：

- 阅读项目交接文件和 AI 指令。
- 检查当前 git 状态。
- 检查框架、路由、样式、运行命令。
- Next.js 项目如有版本特定文档，先读对应文档。

按薄切片实施：

1. schema、seed、配置。
2. 数据访问和 API。
3. 页面框架和导航。
4. 首页看板或列表页。
5. 导入或手工录入。
6. 详情和跟进流程。
7. 权限和后台设置。
8. 可选模块。

避免无关大重构。

## 7. 后台管理补充

第一版也要考虑：

- 初始管理员账号。
- 用户禁用和重置密码。
- 角色权限。
- 导入日志和错误行。
- 操作审计。
- 导出权限。
- 备份和恢复说明。
- 健康检查。

## 8. 最低验收标准

一个学习后台至少要让操作者能：

- 用正确角色登录。
- 查看当前学员状态。
- 知道今天谁需要处理。
- 查看单个学员完整进度。
- 更新或导入进度。
- 记录跟进或下一步动作。
- 判断数据是否新鲜。
- 管理用户、角色和基础配置。

报表、通知、预约、评分、证书等模块启用时，分别补充验收标准。
