# 00-先读我：整体路线

这套方法论适合想用 AI 制作培训进度跟进网站的人。你不需要先学完整编程，也不需要自己读完所有文档。更推荐的方式是：把本仓库交给 AI，让 AI 读取资料、追问需求、生成方案、写代码和做检查；人只确认业务范围、隐私边界和上线决策。

## 这个网站解决什么问题

培训进度跟进网站通常解决这些问题：

- 管理员不知道哪些学员已经完成培训。
- 培训负责人每天要手工统计 Excel。
- 学员进度落后时没有及时提醒。
- 考试、作业、证书、预约记录分散在不同表格里。
- 每周做汇报时需要重复复制、筛选、汇总数据。
- 新项目想复制旧项目经验，但不知道从哪里开始。

网站不是宣传页，而是后台管理工具。第一屏应该直接看到数据、风险和待办，而不是大标题和装饰图。

## 适合的培训项目

| 场景 | 典型需求 |
|---|---|
| 新员工培训 | 入职任务、课程完成率、考试、导师跟进、试用期评估 |
| 3PP 培训 | 外部合作方人员、认证、合规培训、证书到期、复训提醒 |
| 经销商培训 | 区域、门店、人员进度、产品考试、服务标准评分 |
| 岗位认证 | 阶段任务、实操评分、证书状态、未通过补考 |
| 讲师培养 | 学习阶段、试讲预约、评分量表、反馈和改进计划 |

## 为什么不能一开始就写代码

如果你直接对 AI 说“帮我做一个培训网站”，AI 常见问题是：

- 做成宣传首页，不是后台工具。
- 页面好看，但没有真实业务流程。
- 数据字段不完整，后面导入 Excel 时全要重改。
- 没有权限设计，所有人都能看所有数据。
- 没有导入预览和错误提示，真实数据一导入就乱。
- 没有维护、备份、部署说明，上线后没人敢用。

正确做法是让 AI 先问需求，再确认页面和数据，最后写代码。

## 人需要准备什么

人只需要准备这些资料，其他整理工作交给 AI：

- 培训项目名称和目标。
- 学员名单字段：姓名、邮箱、手机号、部门、岗位、批次等。
- 培训阶段和任务清单。
- 进度状态有哪些：未开始、进行中、已完成、逾期、未通过等。
- 管理角色：管理员、培训师、导师、学员、经理。
- 常用报表：周报、月报、完成率、风险名单。
- 如果已有 Excel / CSV，准备一份脱敏样例。

脱敏意思是把真实姓名、手机号、邮箱、公司敏感信息替换成假数据。

## 让 AI 代执行

把这段复制给能读文件或操作项目的 AI：

```text
请根据这个方法论仓库帮我执行培训进度后台项目。

你负责：
1. 读取 AI_CONTEXT.md 和 AI_MANIFEST.yaml
2. 检查目标项目现有文件、运行命令、样例数据和交接规则
3. 向我最多追问 5 个影响范围、权限、导入和部署的关键问题
4. 输出第一版范围、页面、数据字段、权限、导入、API、脚本、验证、部署计划
5. 在我确认后分阶段实施，并运行可用检查

我只负责：
1. 提供业务目标和脱敏样例数据
2. 确认第一版范围
3. 授权账号、部署和生产数据相关操作
```

## 最小可行版本

第一版不要追求所有功能。建议先做：

- 登录
- 首页看板
- 学员列表
- 学员详情
- 阶段 / 任务
- 进度记录
- Excel 导入预览
- 跟进记录
- 基础报表

预约、评分、邮件通知、证书到期、自动周报可以第二阶段再做，除非它们是项目核心。
