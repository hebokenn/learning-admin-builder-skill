# 11-案例：新员工培训和 3PP 培训

这一章用两个例子说明怎么把方法迁移到不同培训项目。

## 案例一：新员工培训

### 业务特点

新员工培训通常关注：

- 是否完成入职材料。
- 是否完成制度课程。
- 是否通过考试。
- 是否完成导师面谈。
- 是否进入岗位试用。

### 页面映射

| 页面 | 用法 |
|---|---|
| 首页看板 | 本批新员工人数、完成率、逾期任务、未通过考试人数 |
| 学员列表 | 按部门、岗位、入职批次查看新人状态 |
| 学员详情 | 查看某个新人的任务、课程、考试、导师反馈 |
| 课程阶段 | 入职准备、公司文化、制度学习、岗位技能、试用期评估 |
| 任务看板 | 合同提交、账号开通、制度考试、导师面谈 |
| 导入页 | 导入 HR 提供的新员工名单 |
| 周报页 | 本周新人培训进度和需要 HR / 导师跟进的人 |
| 设置页 | 配置部门、岗位、导师、提醒规则 |

### 推荐字段

```text
姓名
工号
邮箱
手机号
部门
岗位
入职日期
培训批次
导师
当前阶段
完成率
考试结果
风险状态
备注
```

### 第一版范围

- 学员名单导入。
- 阶段任务跟进。
- 逾期和高风险看板。
- 学员详情。
- 周报导出。

第二阶段再做导师预约、考试评分、邮件自动提醒。

## 案例二：3PP 培训

3PP 可以理解为第三方合作伙伴、外部服务商、渠道人员或合作门店人员培训。

### 业务特点

3PP 培训通常关注：

- 外部组织或合作方。
- 人员准入培训。
- 合规和服务标准。
- 考试认证。
- 证书有效期和复训。
- 不同合作方的数据隔离。

### 页面映射

| 页面 | 用法 |
|---|---|
| 首页看板 | 合作方数量、参训人数、认证通过率、高风险供应商 |
| 学员列表 | 按合作方、区域、角色、认证状态筛选 |
| 学员详情 | 课程进度、考试成绩、证书状态、违规记录 |
| 课程阶段 | 准入培训、产品培训、服务流程、合规考试、认证复训 |
| 任务看板 | 资料提交、线上课程、考试、证书更新 |
| 导入页 | 导入合作方人员名单、考试成绩、认证状态 |
| 周报页 | 各合作方完成率、逾期人员、证书即将过期人员 |
| 设置页 | 合作方、区域、认证有效期、提醒周期 |
| 可选预约 | 线下培训、补考、复训场次 |
| 可选评分 | 考试分数、服务演练评分、认证结果 |

### 推荐字段

```text
姓名
邮箱
手机号
合作方
区域
角色
培训项目
批次
当前阶段
完成率
考试分数
认证状态
证书编号
证书到期日
负责人
备注
```

### 数据模型补充

如果要长期管理认证，建议增加：

```text
organizations：合作方
certifications：证书和有效期
```

如果只是一次性培训，可以先用：

```text
learners + programs + progress + reports
```

## 两个案例的共通方法

不管项目是什么，都按这个顺序：

```text
谁参加培训
谁负责管理
培训分几个阶段
每个阶段有哪些任务
如何判断完成
谁需要被提醒
谁能看到结果
最终如何形成报表
```

这些问题回答清楚后，页面、数据表、API 和脚本都会自然出现。
