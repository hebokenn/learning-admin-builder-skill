# 13-通用 Agent 执行指南

这一章给 WorkBuddy、Kun、Cursor、ChatGPT Agent、Codex 或其他能读文件/改代码的 AI 使用。目标是减少人要读的内容和手动操作，让 AI 尽量代执行。

## 人只负责什么

| 人负责 | 为什么不能完全交给 AI |
|---|---|
| 说明业务目标 | AI 不知道真实业务优先级 |
| 提供脱敏样例数据 | AI 不能凭空知道真实字段 |
| 确认第一版范围 | 避免 AI 做太大 |
| 授权账号、费用、部署和生产密钥 | 涉及安全和成本 |
| 最终验收是否符合业务 | 只有业务负责人能判断 |

其他事情优先让 AI 做。

## AI 负责什么

| AI 负责 | 具体动作 |
|---|---|
| 读取资料 | README、AGENTS、handoff、样例表格、现有代码 |
| 问需求 | 每次最多 5 个关键问题 |
| 整理方案 | 页面、数据、权限、导入、API、脚本、部署 |
| 生成代码 | 按目标项目现有技术栈分阶段实现 |
| 检查质量 | lint、build、测试、浏览器、响应式、安全扫描 |
| 写交接 | 更新目标项目要求的 handoff 或维护记录 |

## 通用启动方式

启动话术统一维护在 [ONE_CLICK.md](../ONE_CLICK.md)。需要给任意 AI 工具“补水”时，优先复制那里的提示词；如果同事能运行命令，直接执行：

```bash
python3 scripts/one_click_context.py --copy
```

## 推荐执行阶段

```text
阶段 0：读取规则和现状
阶段 1：需求澄清和范围确认
阶段 2：页面、UI、数据、权限方案
阶段 3：项目骨架或现有项目适配
阶段 4：核心页面和假数据闭环
阶段 5：数据库、API、导入预览
阶段 6：权限、后台设置、日志
阶段 7：报表、通知、预约、评分等可选模块
阶段 8：本地验证和浏览器检查
阶段 9：部署前准备和交接
```

## 阶段 0：AI 读取现状

AI 应执行：

- 查找 `README.md`、`AGENTS.md`、`docs/AI_HANDOFF.md`、`.workbuddy/skills/`。
- 查找 `package.json`、框架、数据库、运行脚本。
- 查找 `data/`、`samples/`、`uploads/`、Excel / CSV 样例。
- 检查当前 git 状态，避免覆盖用户改动。
- 检查是否已有设计系统和 UI 组件。

AI 输出：

```text
我已读取：
- 项目规则：
- 技术栈：
- 可用命令：
- 已有页面：
- 样例数据：
- 还缺：
```

## 阶段 1：AI 问最少问题

AI 只问会改变实现的问题：

- 唯一学员键是什么。
- 第一版是否必须登录。
- 哪些角色能导入和导出。
- Excel / CSV 是新增、更新、覆盖还是追加。
- 预约、评分、通知、证书是否第一版必须。

不要问：

- 文件里已经写清楚的内容。
- 可以先用默认值的小细节。
- 不影响第一版的数据展示细节。

## 阶段 2：AI 输出可执行方案

方案必须包含：

- 第一版模块。
- 延期模块。
- 页面清单。
- 数据表和字段。
- 权限矩阵。
- 导入流程。
- API 清单。
- 自动化脚本。
- UI 方向。
- 验收命令和页面。
- 部署建议。

可以使用这些模板：

- `templates/AI执行任务单.md`
- `templates/AI执行前项目审查清单.md`
- `templates/表格字段分析报告.md`
- `templates/页面确认清单.md`
- `templates/数据字段清单.md`
- `templates/权限矩阵.md`
- `templates/API与脚本清单.md`
- `templates/API规格模板.md`
- `templates/自动化脚本运行手册.md`
- `templates/本地验证报告.md`
- `templates/部署前Go-No-Go检查表.md`

## 阶段 3 到 9：AI 分阶段执行

AI 每阶段都要：

1. 说明将改哪些文件或模块。
2. 实施小范围改动。
3. 运行可用检查。
4. 记录结果。
5. 再进入下一阶段。

如果遇到生产账号、真实数据、费用、域名、密钥、删除数据、正式发送邮件，必须停下来让人授权。

## 不同工具怎么放上下文

| 工具类型 | 推荐放法 |
|---|---|
| 支持项目规则 | 放 `AGENTS.md`、`AI_CONTEXT.md`、`AI_MANIFEST.yaml` |
| 支持知识库 | 放 `ONE_CLICK.md`、AI_CONTEXT、AI_MANIFEST、templates |
| 支持 skill | 放 `learning-admin-builder/` 整个目录 |
| 只支持一段提示词 | 复制 `ONE_CLICK.md` 的启动语；需要完整独立规则时复制 `portable-prompt.md` |
| 能操作代码仓库 | 让它先读取项目现状并运行检查 |
| 只能聊天 | 让它先产出需求、页面、字段、权限方案 |

## 完成定义

AI 不能只说“已完成”。必须给出证据：

- 改了哪些文件。
- 运行了哪些命令。
- 看了哪些页面。
- 哪些检查通过。
- 哪些操作需要人授权。
- 是否确认没有真实数据、密钥、数据库或私有导出进入仓库。
