# 10-后台管理和日常维护：上线后怎么管

后台管理设计经常被忽略。一个网站能上线，不代表能长期稳定使用。

## 后台管理必须考虑的模块

| 模块 | 作用 |
|---|---|
| 用户管理 | 创建账号、禁用账号、重置密码 |
| 角色权限 | 管理员、培训师、学员、经理能做什么 |
| 学员管理 | 新增、编辑、归档、批量导入 |
| 课程管理 | 阶段、任务、顺序、截止天数 |
| 导入管理 | 上传、预览、错误、日志、回滚 |
| 跟进管理 | 创建、分配、关闭跟进事项 |
| 通知管理 | 模板、发送记录、失败重发 |
| 报表管理 | 周报、月报、导出、发送历史 |
| 系统设置 | 字典、状态、风险规则、邮件配置 |
| 日志审计 | 谁在什么时候改了什么 |
| 备份恢复 | 数据库备份、恢复演练 |
| 健康检查 | 网站、数据库、邮件、脚本是否正常 |

## 管理员工作台

管理员每天打开后台，应该先看到：

- 今日待办
- 高风险学员
- 待跟进事项
- 逾期任务
- 待审批或待评分
- 最近导入状态
- 自动任务是否成功
- 系统健康状态

工作台示例：

```text
今日待办：12
高风险学员：4
逾期任务：18
待评分：6
最近导入：成功，120 行
昨日提醒：已发送 38 封，失败 2 封
备份状态：今日 03:00 成功
```

## 数据维护

### 手动维护

适合：

- 单个学员改部门。
- 给某个学员加备注。
- 标记一个任务已完成。
- 修改一个截止日期。

要求：

- 记录谁改的。
- 记录修改前后。
- 重要字段二次确认。

### 批量导入

适合：

- 新增一批学员。
- 更新考试成绩。
- 同步课程完成情况。

要求：

- 先预览。
- 错误行不导入。
- 冲突行人工确认。
- 导入后能查看日志。

### 误操作恢复

必须考虑：

- 删除能否撤销。
- 导入能否回滚。
- 是否有历史版本。
- 数据库是否有备份。

## 权限和安全

| 角色 | 推荐权限 |
|---|---|
| 超级管理员 | 所有权限 |
| 培训管理员 | 管理负责项目和学员 |
| 培训师 / 导师 | 管理自己负责的学员和评分 |
| 学员 | 只看自己的进度和反馈 |
| 经理 | 看团队汇总和风险，不改课程结构 |
| 只读用户 | 只看报表 |

安全要求：

- 无权限用户看不到按钮。
- 直接访问无权限页面要拦截。
- 敏感字段不要随便导出。
- 导出报表要记录操作人。
- 管理员账号不能共用。

## 日常维护动作

| 动作 | 频率 | 谁负责 |
|---|---|---|
| 检查高风险学员 | 每天 | 培训管理员 |
| 检查导入失败 | 每次导入后 | 导入负责人 |
| 发送逾期提醒 | 每天或每周 | 系统自动 + 管理员 |
| 生成周报 | 每周 | 系统自动 + 管理员确认 |
| 检查备份 | 每周 | 技术负责人 |
| 检查权限账号 | 每月 | 超级管理员 |
| 恢复演练 | 每季度 | 技术负责人 |

## 让 AI 生成维护记录

维护不应完全靠人手写。系统应保留导入日志、通知日志、备份状态、健康检查结果；AI 可以定期读取这些记录，生成维护记录草稿。

给 AI 的提示词：

```text
请根据系统日志和最近变更，生成一份培训后台维护记录草稿。

请检查：
1. 最近导入是否成功，失败行有哪些
2. 最近通知是否发送成功
3. 备份任务是否成功
4. 是否有登录、权限、API 错误
5. 是否有高风险学员或逾期任务异常增加

请输出：
- 本周做了什么
- 风险和异常
- 建议下周处理事项
- 是否需要人工确认
```

## 后台验收清单

- [ ] 管理员能看到今天要处理什么。
- [ ] 能新增、编辑、归档学员。
- [ ] 能导入数据，并看到错误行。
- [ ] 能管理课程阶段和任务。
- [ ] 能记录跟进事项和关闭事项。
- [ ] 能导出报表。
- [ ] 能查看通知发送记录。
- [ ] 能管理用户和角色。
- [ ] 关键操作有日志。
- [ ] 有备份和恢复说明。
- [ ] 权限错误时不会泄露数据。

## 给 AI 的提示词

```text
请帮我设计培训进度跟进网站的后台管理功能。

请重点考虑：
1. 管理员每天打开后台要先看到什么
2. 用户和角色怎么管理
3. 学员、课程、任务怎么维护
4. Excel 导入错误怎么处理
5. 通知和周报怎么记录
6. 谁能导出敏感数据
7. 误操作怎么恢复
8. 需要哪些日志和备份

请输出后台菜单、每个菜单的用途、角色权限、验收清单。
```
