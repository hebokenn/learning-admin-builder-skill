# 04-页面和 UI 设计：后台管理页面怎么设计

培训进度跟进网站应该像工作台，不是宣传网站。设计重点是让管理员快速知道：谁落后、谁需要跟进、今天要做什么。

## UI 设计原则

| 原则 | 说明 |
|---|---|
| 后台工具优先 | 第一屏直接显示数据、风险、待办 |
| 信息密集 | 多用表格、筛选、指标卡，减少装饰 |
| 克制稳定 | 浅色背景、低饱和度、统一按钮和标签 |
| 可扫描 | 状态和风险一眼能看出来 |
| 重复操作友好 | 搜索、筛选、批量提醒、导出要好找 |
| 移动端可读 | 手机端改成卡片，不强行塞复杂表格 |

避免：

- 大面积 hero 区
- 营销口号
- 过多装饰插画
- 夸张渐变
- 每个模块一种颜色

## 页面结构

一个典型后台页面：

```text
顶部导航
页面标题 + 简短说明 + 主要按钮
指标卡
筛选区
主表格 / 图表 / 操作队列
弹窗 / 空状态 / 加载状态
```

## 常见页面

| 页面 | 目的 | 核心信息 | 常见按钮 | 筛选项 |
|---|---|---|---|---|
| 登录页 | 验证身份 | 邮箱、密码 | 登录、忘记密码 | 无 |
| 首页看板 | 看整体状态 | 完成率、逾期、高风险、待跟进 | 导出、提醒 | 项目、批次、部门 |
| 学员列表 | 找人和批量操作 | 姓名、部门、阶段、完成率、风险 | 新增、导入、导出、提醒 | 部门、阶段、状态、导师 |
| 学员详情 | 看单个学员全貌 | 基本信息、任务、成绩、跟进 | 编辑、提醒、添加备注 | 时间、任务类型 |
| 课程结构 | 定义阶段和任务 | 阶段、任务、顺序、天数 | 新增、排序、启用 | 项目、岗位 |
| 任务看板 | 跟踪任务 | 任务、学员、截止、状态 | 批量提醒、标记完成 | 状态、截止日期 |
| 导入页 | 上传 Excel / CSV | 文件、字段、预览、错误 | 上传、预览、确认导入 | 数据类型 |
| 周报页 | 汇总行动 | 完成、逾期、风险、负责人 | 生成、复制、导出 | 周期、部门 |
| 设置页 | 管理后台配置 | 用户、角色、字段、通知规则 | 新增、保存 | 角色、项目 |
| 可选预约页 | 管理面谈/考试 | 时间、导师、学员、状态 | 新建、改期、取消 | 日期、导师 |
| 可选评分页 | 管理考核 | 分数、评级、评语、评分人 | 录入、发布、重评 | 课程、状态 |

## 状态颜色

| 状态 | 含义 | 建议颜色 |
|---|---|---|
| 正常 | 按计划进行 | 蓝色或灰蓝色 |
| 落后 | 低于计划 | 橙色 |
| 高风险 | 可能无法按时完成 | 红色 |
| 已完成 | 完成培训 | 绿色 |
| 待处理 | 需要管理员处理 | 紫色或蓝色 |
| 失败 / 未通过 | 操作失败、考试失败 | 红色 |
| 通过 | 考试通过 | 绿色 |

红色只用于真正重要的风险，不要到处用红色。

## 组件建议

| 组件 | 用途 | 注意点 |
|---|---|---|
| 指标卡 | 展示核心数字 | 每张卡只放一个重点数字 |
| 筛选区 | 反复查询 | 常用筛选直接显示，次要筛选放更多 |
| 表格 | 管理大量学员和任务 | 支持搜索、排序、分页、批量操作 |
| 状态标签 | 快速识别状态 | 颜色全站一致 |
| 进度条 | 展示完成率 | 同时显示数字 |
| 图表 | 展示趋势和分布 | 图表旁保留表格入口 |
| 弹窗 | 确认重要操作 | 按钮文案写具体动作 |
| 空状态 | 没有数据时提示 | 告诉用户下一步做什么 |
| 加载状态 | 避免误以为页面坏了 | 表格和卡片使用占位 |
| 通知中心 | 站内提醒 | 记录已读、未读、发送失败 |
| 日历 | 课程、考试、截止日期 | 点击日期看到当天事项 |

## 给 AI 的 UI 提示词

```text
请为一个“培训进度跟进网站”设计后台管理页面。

目标用户是不懂技术的培训管理员，他们每天需要查看学员培训进度、发现落后人员、发送提醒、导出报表。

请使用后台工具风格，不要做营销页。整体 UI 要密集、克制、可扫描、适合重复操作。不要使用大面积 hero、宣传语、装饰性插画或夸张渐变。

页面需要包含：
1. 顶部导航：进度总览、学员管理、课程管理、任务看板、报表、导入、设置
2. 页面标题区：标题、简短说明、主要操作按钮
3. 指标卡：总学员数、平均完成率、未完成人数、落后人数、高风险人数
4. 筛选区：搜索、部门、课程、状态、风险等级、截止日期、重置筛选
5. 表格：姓名、部门、课程、完成率、最近学习、截止日期、状态、风险、考试结果、操作
6. 状态标签：正常、落后、高风险、已完成、待处理、通过、未通过
7. 加载状态、空状态、错误状态
8. 移动端时表格改为卡片列表
```

## UI 验收清单

- [ ] 第一屏能看出这是培训进度后台。
- [ ] 第一屏能看到关键指标。
- [ ] 管理员能快速找到未完成、落后、高风险人员。
- [ ] 有清晰导航、标题、指标卡、筛选区、表格。
- [ ] 操作按钮文字具体，例如“发送提醒”，不要只写“确定”。
- [ ] 正常、落后、高风险、完成状态清楚。
- [ ] 手机端不强行显示复杂表格。
- [ ] 弹窗不会超出屏幕。
- [ ] 长姓名、长课程名不会重叠。
- [ ] 空状态告诉用户下一步做什么。
