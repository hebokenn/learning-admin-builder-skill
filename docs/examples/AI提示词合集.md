# AI 提示词合集

这些提示词可以直接复制给 Codex、ChatGPT、WorkBuddy、Kun、Cursor 等工具使用。使用前把【】里的内容替换成你的项目资料。

## 1. 让 AI 先问需求

```text
我想做一个培训进度跟进网站，但我是小白。
请你先不要写代码，先像产品经理一样问我问题。

每次最多问 5 个问题。问题要覆盖：
1. 项目目标
2. 使用角色
3. 学员数据来源
4. 培训阶段和任务
5. 报表、提醒和日常管理动作

等我回答后，你继续追问，直到你能整理出完整需求文档。
```

## 2. 让 AI 整理需求文档

```text
请根据刚才的对话，整理《培训进度跟进网站需求文档》。

请包含：
1. 项目目标
2. 用户角色和权限
3. 第一版必须做的功能
4. 第二阶段再做的功能
5. 页面清单
6. 每个页面的核心字段、按钮、筛选项
7. 数据表和字段草案
8. Excel / CSV 导入规则
9. 报表、通知、后台管理要求
10. 验收标准

请用中文，适合不懂技术的人阅读。
```

## 3. 让 AI 设计页面

```text
请为这个培训进度跟进网站设计后台页面。

要求：
- 不要做营销页。
- 第一屏显示指标、风险、待办、最近导入状态。
- 风格要信息密集、克制、可扫描，适合管理员每天重复使用。
- 每个页面都要说明目标、字段、按钮、筛选项、空状态、错误状态、移动端处理。

项目资料：
【粘贴需求文档】
```

## 4. 让 AI 设计数据模型和导入

```text
请根据下面的脱敏 Excel 字段，设计数据模型和导入规则。

字段和样例：
【粘贴字段名和 3-5 行假数据】

请输出：
1. 数据表
2. 字段映射
3. 必填字段
4. 唯一键
5. 状态值标准化
6. 冲突处理
7. 导入预览页面显示什么
8. 导入错误如何记录

先不要写代码。
```

## 5. 让 AI 设计 API 和脚本

```text
请为这个培训进度后台设计 API 和自动化脚本。

请输出：
1. 页面动作到 API 的映射
2. 每个 API 的权限、输入、输出、错误
3. 导入预览和正式导入流程
4. 每日提醒脚本
5. 周报脚本
6. 备份脚本
7. 健康检查脚本

要求不要把密钥写死在代码里，导入必须支持 dry-run 或预览。
```

## 6. 让 AI 判断部署方式

```text
我已经做好培训进度跟进网站，想部署上线。
请读取项目结构和 package.json，判断更适合 Vercel、Render 还是 VPS。

请说明：
1. 推荐平台
2. 为什么推荐
3. 部署前检查项
4. 环境变量清单
5. 数据库选择
6. 备份方案
7. 部署后必须验证的页面和 API
```

## 7. 让 AI 做实现前检查

```text
准备开始实现前，请先检查目标项目。

请你：
1. 阅读项目说明、AGENTS、handoff 或记忆文件
2. 检查现有技术栈、目录、运行命令
3. 检查是否有未提交改动
4. 判断要遵守的 UI 风格和代码风格
5. 输出第一阶段最小改动计划

不要直接大改代码。
```

## 8. 让 AI 做验收

```text
请验收这个培训进度后台。

请检查：
1. 登录、首页、学员列表、学员详情、任务、导入、报表、设置
2. 角色权限是否正确
3. Excel / CSV 导入是否有预览、错误行、冲突提示、日志
4. 1280、1100、900、760、560 宽度是否可用
5. 是否有备份、日志、导出权限、维护说明
6. 是否误包含真实数据、密钥、数据库或生产配置

请输出问题清单、严重程度、建议修复顺序。
```
