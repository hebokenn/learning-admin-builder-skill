# 03-让 AI 问需求：不要一上来就写代码

AI 做网站最容易失败的原因，不是不会写代码，而是需求没问清楚。

你要先让 AI 像产品经理一样问你问题，再让它整理需求文档。人不需要自己写完整 PRD，只需要回答 AI 问的关键问题，并确认第一版范围。

## 最少人工流程

```text
把项目资料交给 AI
  -> AI 检查资料并提问
  -> 人回答关键问题
  -> AI 整理需求文档和第一版范围
  -> 人确认
  -> AI 再开始页面、数据、代码和验证
```

## 第一步：让 AI 主动问问题

复制这段给 AI：

```text
我想做一个培训进度跟进网站，但我是小白。
请你先不要写代码，先像产品经理一样问我问题。

每次最多问 5 个问题，问题要围绕：
1. 培训项目是什么
2. 谁会使用这个网站
3. 要管理哪些学员信息
4. 要跟踪哪些学习进度
5. 要做哪些统计报表
6. 哪些操作每天都会重复

等我回答后，你继续追问，直到你能整理出完整需求文档。
```

如果你已经有项目文件、Excel 或旧文档，用这段：

```text
我会给你一个项目目录或一些资料。请你先读取已有资料，不要重复问文件里已经有的信息。

请你检查：
1. README、AGENTS、handoff 或项目规则
2. 现有页面和代码结构
3. 样例 Excel / CSV 的 sheet、表头和样例行
4. 已有字段、状态、角色、报表

然后只问我最多 5 个仍然缺失、且会影响数据结构、权限、导入或部署的关键问题。
```

## 第二步：让 AI 整理需求文档

等 AI 问完几轮后，复制这段：

```text
请根据刚才的对话，整理一份《培训进度跟进网站需求文档》。

请包含：
1. 项目目标
2. 使用角色
3. 每个角色能做什么
4. 页面列表
5. 每个页面的功能
6. 需要保存的数据字段
7. 常用操作流程
8. 第一版必须做的功能
9. 以后再做的功能
10. 验收标准

请用中文，适合不懂技术的人阅读。
```

同时让 AI 生成机器可执行摘要：

```text
请在需求文档最后附一段《给 AI 执行用的压缩摘要》。
摘要必须包含：
1. 第一版模块
2. 暂不做模块
3. 页面清单
4. 数据表和字段
5. 权限规则
6. 导入规则
7. 验收命令和页面
```

## 第三步：把功能分成第一版和以后再做

第一版建议只做能跑通闭环的功能：

- 登录
- 学员列表
- 学员详情
- 课程阶段
- 任务进度
- 导入
- 基础看板
- 跟进记录

以后再做：

- 邮件通知
- 自动周报
- 预约
- 评分
- 证书
- 多组织权限
- 第三方系统同步

给 AI 的提示词：

```text
请把我的需求分成“第一版必须做”和“以后再做”。
判断标准：
- 第一版只保留能完成培训进度跟进闭环的功能。
- 暂时不做会显著增加复杂度但不是必须的功能。
- 每个被延期的功能都说明为什么可以以后再做。
```

## 第四步：确认验收标准

验收标准就是判断“做完没有”的清单。

示例：

```text
请为“学员列表页”生成验收标准。
页面目标：管理员可以查看所有学员的培训进度，并找到需要跟进的人。

请输出：
- 能搜索姓名、邮箱或工号
- 能按部门、批次、状态、风险筛选
- 能看到完成率、当前阶段、最近学习时间
- 能点击进入学员详情
- 没有数据时有空状态
- 手机端不溢出
```

## 常见错误

| 错误说法 | 更好的说法 |
|---|---|
| 帮我做一个培训网站 | 先帮我问需求，不要写代码 |
| 做得高级一点 | 做成后台管理工具，信息密集、克制、可扫描 |
| 加一个学员页 | 学员页要显示哪些字段、按钮、筛选、权限 |
| 支持导入 Excel | Excel 有哪些列、哪些必填、冲突怎么处理 |
| 加通知 | 什么时候通知、通知谁、用邮件还是站内信 |
