# 08-本地运行和验收：先在自己电脑上跑通

上线前一定要先本地运行。不要在自己电脑都打不开的情况下直接部署。能操作终端和浏览器的 AI 应该直接执行检查；人只看结果和确认是否继续。

## 让 AI 直接执行本地检查

把这段给能操作项目的 AI：

```text
请帮我本地验证这个培训进度后台。

请你执行：
1. 读取 package.json，确认安装、启动、lint、build、test 命令
2. 安装依赖，但不要升级大版本依赖
3. 启动本地服务，如果端口被占用就换一个端口
4. 打开首页、登录页、学员列表、学员详情、导入、报表、设置
5. 检查 1280、1100、900、760、560 宽度
6. 记录报错、截图路径或页面现象
7. 给出必须修复的问题和可以延后的问题

如果需要账号、验证码、生产数据或外部服务密钥，请先问我。
```

AI 应把结果写成 [本地验证报告](../templates/本地验证报告.md)，报告里必须有命令、结果、页面、证据和阻塞问题。

## AI 自动验收流水线

AI 应按这个顺序执行，某一步失败就记录原因并判断是否继续：

1. 识别包管理器：npm、pnpm、yarn、bun。
2. 读取 `package.json` scripts。
3. 安装依赖，不主动升级大版本。
4. 运行 lint、typecheck、test、build 中可用的命令。
5. 启动本地服务。
6. 打开关键页面。
7. 检查 1280、1100、900、760、560 宽度。
8. 做关键 API smoke test。
9. 做导入 dry-run 或预览测试。
10. 做权限直连 API 测试。
11. 扫描 `.env`、数据库、真实表格、密钥是否误入仓库。

人只需要看 AI 的验证报告，确认阻塞问题是否必须修。

## 本地运行基本流程

通常是：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

不同项目命令可能不同，要以项目 `package.json` 里的 scripts 为准。

## 让 AI 帮你解释命令

```text
我是小白，现在要在本地运行这个培训进度跟进网站。
请你读取 package.json，告诉我：
1. 安装依赖用什么命令
2. 本地启动用什么命令
3. 构建检查用什么命令
4. 成功后浏览器打开哪个地址
5. 如果失败，我应该复制哪些错误给你
```

## 验收页面

第一版至少检查：

- 登录页
- 首页看板
- 学员列表
- 学员详情
- 课程 / 阶段
- 任务看板
- 导入页
- 报表页
- 设置页

如果有可选模块，再检查：

- 预约
- 评分
- 通知中心
- 证书

## 响应式检查

检查这些宽度：

- 1280：常见电脑屏幕
- 1100：小笔记本
- 900：平板横屏
- 760：平板 / 大手机
- 560：手机

要看：

- 是否横向溢出。
- 按钮是否被裁切。
- 长文字是否重叠。
- 弹窗是否超出屏幕。
- 表格在手机上是否改成卡片或可滚动。
- 筛选区是否还能使用。

## 数据验收

准备几组假数据：

| 数据场景 | 为什么要测 |
|---|---|
| 没有学员 | 检查空状态 |
| 1 个学员 | 检查基础详情 |
| 50 个学员 | 检查筛选和分页 |
| 已完成学员 | 检查完成状态 |
| 逾期学员 | 检查风险标签 |
| 未通过考试 | 检查评分和提醒 |
| 导入错误行 | 检查错误提示 |
| 无权限用户 | 检查权限 |

## 常用检查命令

```bash
npm run lint
npm run build
```

如果项目没有这些命令，让 AI 先读取 `package.json` 后再判断。

## 报错时怎么问 AI

```text
我本地运行培训进度网站时报错。

我做了这些操作：
1. 【步骤 1】
2. 【步骤 2】

终端错误：
【粘贴完整错误】

浏览器现象：
【页面空白 / 登录失败 / 数据不显示】

请你判断：
1. 最可能原因
2. 需要我检查哪些文件
3. 下一步只做一个最小修复
```
