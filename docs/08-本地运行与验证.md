# 08-本地运行和验收：先在自己电脑上跑通

上线前一定要先本地运行。不要在自己电脑都打不开的情况下直接部署。

## 本地运行基本流程

通常是：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

不同项目命令可能不同，要以项目 `package.json` 里的 scripts 为准。

## 让 AI 帮你解释命令

```text
我是小白，现在要在本地运行这个培训进度跟进网站。
请你读取 package.json，告诉我：
1. 安装依赖用什么命令
2. 本地启动用什么命令
3. 构建检查用什么命令
4. 成功后浏览器打开哪个地址
5. 如果失败，我应该复制哪些错误给你
```

## 验收页面

第一版至少检查：

- 登录页
- 首页看板
- 学员列表
- 学员详情
- 课程 / 阶段
- 任务看板
- 导入页
- 报表页
- 设置页

如果有可选模块，再检查：

- 预约
- 评分
- 通知中心
- 证书

## 响应式检查

检查这些宽度：

- 1280：常见电脑屏幕
- 1100：小笔记本
- 900：平板横屏
- 760：平板 / 大手机
- 560：手机

要看：

- 是否横向溢出。
- 按钮是否被裁切。
- 长文字是否重叠。
- 弹窗是否超出屏幕。
- 表格在手机上是否改成卡片或可滚动。
- 筛选区是否还能使用。

## 数据验收

准备几组假数据：

| 数据场景 | 为什么要测 |
|---|---|
| 没有学员 | 检查空状态 |
| 1 个学员 | 检查基础详情 |
| 50 个学员 | 检查筛选和分页 |
| 已完成学员 | 检查完成状态 |
| 逾期学员 | 检查风险标签 |
| 未通过考试 | 检查评分和提醒 |
| 导入错误行 | 检查错误提示 |
| 无权限用户 | 检查权限 |

## 常用检查命令

```bash
npm run lint
npm run build
```

如果项目没有这些命令，让 AI 先读取 `package.json` 后再判断。

## 报错时怎么问 AI

```text
我本地运行培训进度网站时报错。

我做了这些操作：
1. 【步骤 1】
2. 【步骤 2】

终端错误：
【粘贴完整错误】

浏览器现象：
【页面空白 / 登录失败 / 数据不显示】

请你判断：
1. 最可能原因
2. 需要我检查哪些文件
3. 下一步只做一个最小修复
```
